# Нагрузочный тест

## Параметры

- Инструмент: Apache Bench (`ab`)
- Длительность: 30 секунд на каждый эндпоинт

## Состояние БД перед тестами

БД сброшена и инициализирована заново командой `make seed`:

| Сущность      | Количество              |
|---------------|-------------------------|
| Пользователей | 10 002                  |
| Переговорок   | 50                      |
| Расписаний    | 50 (пн–вс, 08:00–18:00) |
| Слотов        | 206 000 (~1 000/день)   |
| Бронирований  | 101 581                 |
| active        | 85 803                  |
| cancelled     | 15 778                  |

### POST /register

Команда:
```bash
ab -n 999999 -c 50 -t 30 -p register_body.json -T 'application/json' http://localhost:8080/register
```

> ab шлёт одинаковый email -> 409 после первого запроса. bcrypt выполняется **до** проверки уникальности email, поэтому CPU-нагрузка та же, что при реальной регистрации. Латентность отражает реальную стоимость операции.

### POST /bookings/{id}/cancel

Команда:
```bash
ab -n 999999 -c 100 -t 30 -p empty_body.json -T 'application/json' \
  -H "Authorization: Bearer $USER_TOKEN" "http://localhost:8080/bookings/{bookingId}/cancel"
```

> Idempotent: первый вызов отменяет бронь, все последующие возвращают 200 OK с `status: cancelled` без повторного UPDATE в БД.
